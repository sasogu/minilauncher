## Siguiente checklist

1. Subir el repositorio a GitHub o GitLab en publico.
2. Crear el tag `v0.1.0`.
3. Revisar si quieres mantener `Apache-2.0` o cambiar a otra licencia FLOSS.
4. Capturar screenshots definitivas de la app.
5. Preparar icono promocional si quieres mejorar la ficha.
6. Abrir una solicitud en `fdroiddata` con la metadata del proyecto.

## Datos que normalmente pide F-Droid

- URL del repositorio
- licencia
- nombre de la app
- `applicationId`
- instrucciones de build
